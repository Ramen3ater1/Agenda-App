# UI 更改报告生成规范

## 核心职责
每当您对UI目录下的任何TSX文件进行修改后，**必须**生成一份详细的更改报告。

## 报告要求

### 1. 报告结构
每次更改后，请按以下格式输出报告：

```markdown
## 📋 UI 更改报告 - [YYYY-MM-DD HH:MM:SS]

### 📁 更改文件清单
- `path/to/component1.tsx` - [简要说明更改目的]
- `path/to/component2.tsx` - [简要说明更改目的]

---

### 📄 文件详情：[文件名].tsx

#### 更改的模块/组件
- **组件名称**: [组件函数名或类名]
- **更改位置**: [第X行 - 第Y行]
- 对应的ui部分: 通俗描述修改对应的视觉组件
#### 更改的代码片段
```tsx
// 更改前
[原始代码]

// 更改后
[新代码]
